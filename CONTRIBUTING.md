# Contributing

欢迎提交 Issue 和 Pull Request。

## 提交建议

- Bug 修复请尽量提供复现步骤
- 站点适配建议附带目标页面的 DOM 特征
- 若是识别错误，请说明：
  - 哪些内容被误抓
  - 哪些内容漏抓
  - 目标站点名称

## 开发流程建议

1. Fork 仓库
2. 创建功能分支
3. 完成修改并本地测试
4. 更新相关文档
5. 提交 Pull Request

## 提交前检查

- 保持功能改动与需求一致
- 避免引入与当前任务无关的代码
- 确认没有误修改打包文件内容
- 如变更识别逻辑，请至少说明适配了哪些站点

## 代码风格

- 变量和函数命名保持语义清晰
- DOM 识别逻辑优先使用小范围选择器
- 样式类名前缀统一使用 `aics-`
