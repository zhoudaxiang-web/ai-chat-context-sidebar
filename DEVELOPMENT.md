# Development Guide

## 1. 项目目标

本扩展的目标是在 AI 聊天界面中，为“用户提问内容”生成一个可交互的侧栏导航，帮助用户快速浏览和定位长会话中的历史提问。

## 2. 技术结构

本项目基于 Chrome Extension Manifest V3 构建，同时兼容 Edge。

### 关键文件

- `manifest.json`
  扩展清单文件，定义权限、内容脚本、图标与基础配置
- `content.js`
  扩展核心逻辑，包括：
  - 站点规则适配
  - 用户提问识别
  - 侧栏渲染
  - 点击定位
  - 拖动 / 拉伸 / 折叠交互
- `content.css`
  侧栏样式与交互态样式

## 3. 核心功能模块

### 3.1 站点适配

`content.js` 中维护了 `SITE_RULES`，用于识别不同 AI 网站的聊天容器和用户提问节点。

当前策略：

- 优先使用站点专属选择器
- 若专属选择器不足，则使用通用用户消息选择器
- 个别站点可使用定制回退逻辑，例如豆包

### 3.2 用户提问识别

目标是只识别“用户发起的提问”，不包含：

- AI 回答
- 页面侧边栏历史标题
- 会话总标题
- 系统提示文本

### 3.3 侧栏渲染

侧栏提供：

- 提问预览列表
- 提问点击定位
- 独立滚动
- 展开 / 折叠
- 拖动位置
- 拉伸高度

### 3.4 状态持久化

通过 `chrome.storage.sync` 按站点保存以下信息：

- 手动选择的聊天容器
- 侧栏展开状态
- 侧栏顶部位置
- 侧栏高度

## 4. 本地开发

### 安装依赖

本项目当前不依赖 npm 包，可直接开发。

### 调试方式

1. 在 Chrome / Edge 中加载已解压扩展
2. 修改 `content.js` 或 `content.css`
3. 回到扩展管理页点击“刷新”
4. 刷新目标网页验证效果

## 5. 代码规范

- 保持内容脚本逻辑按功能分段
- 对站点适配优先使用最小范围的选择器
- 尽量避免使用过宽的通用抓取规则误识别 AI 回答
- 不在内容脚本中引入不必要依赖
- 样式命名统一使用 `aics-` 前缀，避免污染页面样式

## 6. 新增站点适配方式

新增站点时，推荐流程：

1. 在 `SITE_RULES` 中新增 host 配置
2. 添加聊天容器选择器
3. 添加用户提问节点选择器
4. 在真实页面中验证：
   - 是否只抓到用户提问
   - 是否误抓回答
   - 是否误抓标题
5. 如有必要，为该站点增加专属回退逻辑

## 7. 发布前检查

发布前建议至少确认以下内容：

- `manifest.json` 配置完整
- 图标文件齐全
- 常见站点能正常识别用户提问
- 折叠、滚动、拖动、拉伸交互无明显问题
- Chrome / Edge 均可正常加载
- `PRIVACY.md` 与实际行为一致

## 8. 已知限制

- 不同 AI 网站 DOM 结构变化较快，站点适配需要持续维护
- 某些站点使用 Shadow DOM 或动态虚拟列表时，可能需要额外定制
- 复杂页面结构下，仍可能需要用户手动选择聊天容器
